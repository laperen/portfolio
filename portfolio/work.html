ICA Border Control Training Application

  • This is a Solo project while working in ST Engineering
  • Made in Unity
  • This is an Eye-tracking training application, for the purpose of training future border control officers.
A promotional video by ICA, also shown here on facebook.

Scripting and Functionality

  • Created a 2D Scenario format, which took note of when subject gazed at a marked area of interest.
  • Data-playback, not video playback, of eye tracking visualizations.
  • Networking between trainer and trainee stations.

Responsibilities

  • Made sure heatmap and eye-tracking visualizations were intuitive for instructors.
  • Editor for creating scenarios by trainers, for trainees, had to be simple to use, yet fulfil complex training needs.
  • Results and review screens must contain all important information available at a glance.

Trials and Tribulations

  • Working alone meant doing everything, including gathering and vetting customer rquirements.
  • The secure nature of the application made netowrking features particularly challenging to implement, requiring a complete overhaul of core functionality while nearly 3/4 of the way into the project.

ASTAR SimTech Virtual Manufacturing Factory

  • This was a 2-man project while working in ASTAR under their SimTech department.
  • Made in Unity
  • This is a multiplayer, multi-platform, manufacturing factory simulator.
  • Use cases included training technicians on maintainence processes, and simulating real world factories for virtual tours
The multiplayer multiplatform capabilities, with 2 players running a training scenario.

Scripting and Functionality

  • Frontend application has Desktop, Mobile, and VR versions.
  • All frontend applications connect to the same server, and can see each other while they are in the same room.
  • Players are able to create new rooms, and join existing rooms.
  • When creating a room, players have a selection of factories to choose to enter.
  • If the room created is for training, players in the same room will be able to complete the scenario they are in collaboratively.
  • If the room create is for visualization, players in the same room will see the same simulation.
  • There is the option to replicate actions of a real factory, but requires that the factory maintains a MQTT broker with messages from the machines flowing through.
An older version of the application, but showcasing a virtual simulation of simple factory operations

Responsibilities

  • Created architecture for multi-platform detection, to spawn in the appropriate player controller object, for the player to interact with the world.
  • The reason to go for multi-platform detection rather than deliberately shipping any one version of the app with a specific player controller, is because Desktop and Mobile can enable VR.
  • Interactable objects have all interaction scripts, to react to all player controller types, allowing designers to only make the scene once, but be usable on all target platforms.
  • The user interaface has to change based on the platform as well, being a screen overlay for PC and Mobile, but going into world space ethereal screens while in VR.

Trials and Tribulations

  • Creating a multiplayer multiplatform interactive suite has been an aspiration of mine. Being given the chance to get paid while doing it was quite a dream come true.
  • Although not my first time dealing with multiplayer, this was the first time I had to deal with object ownership during player interactions, a humbling experience.
  • Designing UI capable of switching between VR and non-VR platforms was a challenge. Although the solution I came up with in this project works, it is ultimately a pain to setup any UI with it.
  • The project ultimately ended up being a pitch piece due to a lack of demand.

SIT Virtual Network Lab

  • This was a 3-man project while working in SIT, in partnership with DigiPen.
  • Made in Unity
  • This is a multi-platform web based teaching aid.
  • A virtual sandbox simulating a Network Lab environment, for players to practice their curriculum.
A small demonstration of features

Scripting and Functionality

  • A flexible interaction system which allows player controllers of multiple platforms to interact with objects, while remaining intuitive to use for the player.
  • Players are able to create network labs, choosing what network objects they want, such as switchers and computers.
  • While in a network lab, the player has the option to enter an editing mode, adding and removing network objects as they please.
  • Network objects can be connected with network cables.
  • Laptop OS was replicated minimally, just enough to simulate network lab applications.

Responsibilities

  • The 3 man team was divided between back-end, front-end, art and design. I was working on the front-end.
  • Created architecture for multi-platform detection, to spawn in the appropriate player controller object, for the player to interact with the world.
  • Multi-platform detection is absolutely necessary, since this is a web application.
  • Interactable objects have all interation scripts, to react to all player controller types at all times, allowing designers to only make the scene once, but be usable on all target platforms.
  • The user interaface has to change based on the platform as well, being a screen overlay for PC and Mobile, but going into world space ethereal screens while in VR.
  • Lab creation and lab editing has to be as easy to use as possible.

Trials and Tribulations

  • I joined the team mid-way in development. Working from a halfway created project was quite a different experience.
  • With my responsibility to work on the front-end, it became my call for what parts of the code had to be removed, and what could be kept to integrate with what I bring in.
  • Initially, the scene setup, object scaling, and player interaction code was a mess, and had to be overhauled.
  • Overall this project was remarkably similar to the ASTAR VMF project, aside from being a web application. I took it as an opportunity to refine part of what I had worked on in the previous project.
  • The player controls for PC and Mobile are much more intuitive even compared to my old work.
  • Setup for switching between screen overlay and world space is nolonger difficult to work with.
